# Project Constitution

This repository uses AEHF in lightweight greenfield mode.

## Non-negotiable rules

1. Prefer clarity over cleverness.
2. Keep scope small and reviewable.
3. Preserve repository memory as the project evolves.
4. Match process depth to task reality.
5. Do not add complexity that the MVP does not need.
6. Distinguish facts from assumptions.
7. Keep docs and implementation aligned.

## Greenfield posture

This project is intentionally small.
The goal is not to prove enterprise scale.
The goal is to validate whether AEHF improves delivery quality without creating unnecessary process.
